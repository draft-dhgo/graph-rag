## 개요 요약

Microsoft GraphRAG는 대규모 언어 모델(LLM)을 사용하여 구조화되지 않은 텍스트에서 구조화된 지식을 추출하는 정교한 데이터 파이프라인 및 변환 도구 모음입니다. 지식 그래프 구성, 커뮤니티 감지, 벡터 임베딩을 결합하여 고급 검색 증강 생성(RAG) 기능을 제공합니다.

### 핵심 특징
- **모듈형 아키텍처**: 명확한 관심사 분리와 잘 정의된 인터페이스
- **다중 제공업체 지원**: OpenAI, Azure OpenAI, LiteLLM (100+ 제공업체)
- **유연한 스토리지**: 로컬 파일 시스템, Azure Blob, Azure Cosmos DB
- **다양한 쿼리 방법**: [[Local Search]], [[Global Search]], [[DRIFT Search]], 기본 검색
- **프로덕션 준비 완료**: 포괄적인 오류 처리, 재시도 로직, 속도 제한

## 주요 기능

### 1. 지식 그래프 구성

**구성 요소:**
- LLM을 사용한 엔티티 추출
- 가중치 계산이 포함된 관계 추출
- 단언(assertion)을 위한 클레임/공변량 추출
- 그래프 프루닝 및 검증

**알고리즘:**
- 커뮤니티 감지를 위한 [[Leiden Algorithm]]
- 그래프 임베딩을 위한 Node2Vec
- 차원 축소를 위한 UMAP

### 2. 쿼리 방법

| 방법 | 최적 용도 | 기술 |
|--------|----------|-----------|
| [[Local Search]] | 구체적인 질문 | 엔티티 중심, 2-3홉 컨텍스트 |
| [[Global Search]] | 광범위한 질문 | 커뮤니티 보고서 집계 |
| [[DRIFT Search]] | 복잡한 다중 홉 | 반복적 정제 |
| 기본 검색 | 간단한 검색 | 전통적인 RAG |

### 3. 파이프라인 아키텍처

**표준 파이프라인 단계:**
1. 기본 [[Text Unit]] 생성 (청킹)
2. 최종 문서 생성
3. 그래프 추출 (엔티티 + 관계)
4. 그래프 최종화 (정리, 검증)
5. 공변량 추출
6. [[Community]] 인스턴스 생성 (Leiden 클러스터링)
7. 최종 텍스트 유닛 생성
8. [[Community Report]] 인스턴스 생성
9. 텍스트 임베딩 생성

## 권장사항

### 도입을 위해
1. **로컬 모드로 시작**: 로컬 스토리지와 OpenAI API 사용
2. **빠른 파이프라인 사용**: 초기 프로토타이핑을 위해
3. **캐싱 활성화**: LLM 비용 상당히 절감
4. **사용자 정의 프롬프트**: 도메인 정확도를 위해 prompt-tune 사용

### 프로덕션을 위해
1. **Azure 통합**: 확장성을 위해 Azure 서비스 사용
2. **모니터링**: 콜백 핸들러 구현
3. **오류 처리**: 적절한 재시도 전략 구성
4. **리소스 관리**: 속도 제한 및 동시 요청 튜닝

## 결론

GraphRAG는 지식 그래프 구성 및 검색을 위한 잘 설계된 프로덕션 준비 시스템입니다. 모듈형 설계, 광범위한 구성 옵션, 다중 제공업체 지원은 연구 프로토타입부터 엔터프라이즈 배포까지 다양한 애플리케이션에 적합하게 만듭니다.
